## 一、数据集概述

### 1.1 评测维度

本数据集覆盖以下评测维度：

| 维度 | 说明 |
|-----|------|
| **检索质量** | 系统能否准确检索到与问题相关的知识库内容 |
| **回答质量** | 系统生成的回答是否准确、完整、有用 |
| **置信度准确性** | 置信度分值是否与实际回答质量匹配 |
| **分流准确性** | 高/中/低置信度的分流是否符合预期 |
| **边界 case** | 系统对模糊、歧义、无关问题的处理能力 |

### 1.2 知识库覆盖

本数据集基于以下知识库文档构建：

| 文档编号 | 文档名称 | 覆盖场景 |
|---------|---------|---------|
| IT-HR-001 | 员工入职离职IT账号处理指南 | 入职、离职、权限申请、门禁 |
| IT-NET-002 | 公司网络问题排查手册 | WiFi、VPN、网络连接、排查命令 |
| IT-PRINT-003 | 打印机故障快速解决指南 | 打印故障、卡纸、驱动、队列 |
| IT-SOFT-004 | 软件安装与授权申请指南 | 软件安装、授权申请、分类 |
| IT-SEC-005 | 账号密码与信息安全手册 | 密码重置、钓鱼防范、安全规范 |

### 3.2 回答质量标准

对于每个 Query，定义"好回答"和"坏回答"的标准：

#### 示例格式

```json
{
  "query_id": "PRINT-ERR-002",
  "query": "打印机卡纸了怎么取出",
  "good_answer特征": [
    "包含'立即关闭打印机电源'",
    "说明不同位置卡纸的取出方法（纸盒、出纸口、内部）",
    "包含检查纸张完整的步骤",
    "提到重新装纸并开机测试",
    "包含预防卡纸的建议（如80g纸张、不要超过容量）"
  ],
  "bad_answer特征": [
    "仅说'重启打印机'，无具体卡纸取出步骤",
    "缺少安全提示（如不断电取纸可能造成人身伤害）",
    "未提及不同卡纸位置的解决方法",
    "未提及预防措施"
  ],
  "minimum_required_steps": [
    "关闭打印机电源",
    "根据位置取出纸张",
    "检查纸张完整",
    "重新装纸开机测试"
  ]
```

## 四、Expected Behavior（期望行为）

### 4.1 置信度分级标准

| 置信度范围 | 期望行为 | 触发条件 |
|-----------|---------|---------|
| **≥ 0.8** | 直接展示 AI 回答 | 检索到高相关内容 + 回答结构完整 |
| **0.6 - 0.8** | 展示回答 + 转人工建议 | 检索到相关内容但有歧义/多解 |
| **< 0.6** | 不展示回答 + 转人工创建工单 | 检索结果不相关 / 知识库无覆盖 / 歧义问题 |

### 4.2 分流准确性判定

```json

  "query_id": "NET-WIFI-001",
```

## 五、评测指标定义

### 5.1 检索指标

| 指标 | 计算方式 | 目标值 |
|-----|---------|-------|
| **Recall@K** | 正确答案出现在 Top-K 结果的比例 | > 85% |
| **MRR** | 平均倒数排名 (1/rank 的平均值) | > 0.75 |
| **NDCG@K** | 考虑排名权重的归一化增益 | > 0.70 |

### 5.2 生成指标

| 指标 | 计算方式 | 目标值 |
|-----|---------|-------|
| **回答正确率** | 人工标注为"正确"的比例 | > 80% |
| **幻觉率** | 回答包含知识库没有的内容的比例 | < 10% |
| **关键词覆盖率** | 回答包含知识库关键信息的比例 | > 70% |

### 5.3 业务指标

| 指标 | 计算方式 | 目标值 |
|-----|---------|-------|
| **分流准确率** | 置信度判定与人工判定一致的比例 | > 80% |
| **AI直接解决率** | 置信度≥0.6 且员工未转人工的比例 | > 65% |
| **员工满意度** | 员工对AI回答的评分（1-5） | > 4.0 |

## 六、测试执行规范

### 6.1 测试集划分

| 划分 | 数量 | 用途 |
|-----|------|------|
| **基础测试集** | 40条 | 覆盖高频基础场景 |
| **中等测试集** | 25条 | 覆盖中等难度场景 |
| **边界测试集** | 15条 | 测试边界 case |
| **总计** | 80条 | 全面覆盖 |

### 6.2 评测流程

```
1. 系统准备
   - 知识库已同步到最新版本
   - ChromaDB 集合已重建
   - 系统运行正常

2. 执行评测
   - 对每个 Query 执行检索
   - 记录 Top-5 检索结果
   - 记录置信度分值
   - 记录分流行为

3. 结果标注
   - 人工判断检索是否相关
   - 人工判断回答是否正确
   - 人工判断分流是否合理

4. 指标计算
   - 计算 Recall@K、MRR、NDCG
   - 计算分流准确率
   - 汇总业务指标

5. 问题分析
   - 找出检索失败的原因
   - 找出回答质量差的原因
   - 提出优化建议
```
