## 1. 入职IT账号申请流程

### 1.1 流程概述

新员工入职时，IT部门需要在**入职前一个工作日**完成所有账号的创建和开通，确保员工入职当天即可正常使用公司IT资源。

### 1.2 具体操作步骤

#### 第一步：接收入职通知单

- HR部门通过邮件发送《入职通知单》至 IT邮箱 (it-support@company.com)
- 通知单包含：姓名、部门、岗位、入职日期、主管姓名
- IT管理员在收到通知后24小时内响应

#### 第二步：创建企业账号

| 账号类型 | 账号格式 | 说明 |
|---------|---------|------|
| 企业邮箱 | zhangsan@company.com | 格式为姓名拼音首字母 |
| OA系统 | zhangs 或 zhangsan | 姓在前，名在后 |
| 钉钉/企业微信 | 手机号注册 | 需确认入职发放的手机号 |
| 门禁卡 | 工牌编号 | 到前台领取 |

#### 第三步：开通业务系统权限

根据员工岗位，确定需要开通的系统权限：

```
┌─────────────────────────────────────────────────────────────┐
│ 基础权限（全员必须）                                          │
├─────────────────────────────────────────────────────────────┤
│ □ 企业邮箱 (@company.com)                                    │
│ □ OA系统 (集团办公自动化)                                     │
│ □ 企业微信/钉钉                                              │
│ □ 打印机访问权限                                             │
│ □ 会议室预订系统                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 业务系统权限（按岗位开通）                                   │
├─────────────────────────────────────────────────────────────┤
│ □ 财务系统（金蝶/用友）- 财务部                              │
│ □ ERP系统 - 运营部/采购部                                   │
│ □ CRM系统 - 销售部/市场部                                   │
│ □ HR系统 - 人力资源部                                       │
│ □ 研发代码库(GitLab) - 研发部                               │
│ □ VPN权限 - 需审批，常出差人员                               │
└─────────────────────────────────────────────────────────────┘
```

#### 第四步：准备设备

- 台式机：提前装机，安装常用软件（Office、钉钉、PDF阅读器）
- 笔记本电脑：加域、配置网络打印机
- 工牌+门禁卡：前台制作，IT确认门禁权限
- 耳机/话机：如是电话岗位，准备耳麦

#### 第五步：发送账号信息

将账号信息通过**加密邮件**发送至：
- 新员工个人邮箱（非公司邮箱）
-抄送：HR负责人、入职员工主管

## 2. 账号开通检查清单

### 2.1 标准检查清单

员工入职当天，IT人员需陪同完成以下检查：

#### ☑️ 账号登录检查

| 检查项 | 状态 | 备注 |
|-------|------|------|
| 企业邮箱可登录，可收发邮件 | □ | 测试发送测试邮件 |
| OA系统可登录，能查看公告 | □ | 检查默认密码是否需要修改 |
| 企业微信/钉钉可正常使用 | □ | 确认已加入公司群 |
| 门禁卡可正常开门 | □ | 测试大门和部门门 |

#### ☑️ 软件安装检查

| 检查项 | 状态 | 备注 |
|-------|------|------|
| Windows/Mac系统正常运行 | □ | 检查最新补丁 |
| Office套件可正常使用 | □ | Word/Excel/PPT |
| 打印机驱动已安装 | □ | 测试打印一页 |
| 杀毒软件已启用 | □ | 确保病毒库最新 |
| 截图软件/常用工具 | □ | Snipaste等 |

#### ☑️ 网络访问检查

| 检查项 | 状态 | 备注 |
|-------|------|------|
| 有线网络可上网 | □ | 测试访问百度 |
| WiFi可连接（如需） | □ | 确认SSID和密码 |
| VPN可连接（如需） | □ | 测试VPN账号 |

## 3. 离职账号处理流程

### 3.1 流程概述

离职账号处理需在**员工最后工作日**完成，确保公司信息安全。

### 3.2 离职操作时间节点

```
离职申请提交（第-5天前）
        ↓
IT账号冻结检查（第-3天）
        ↓
最后工作日账号处理（Day 0）
        ↓
账号正式关闭（第+1天）
        ↓
数据备份与交接（可选，第+3天内）
```

### 3.3 账号处理操作

#### 必须立即执行

| 操作 | 执行人 | 说明 |
|------|--------|------|
| 禁用企业邮箱 | IT管理员 | 邮件保留6个月后可删除 |
| 禁用OA系统账号 | IT管理员 | 禁止登录 |
| 移除钉钉/企微 | IT管理员或部门主管 | 从企业组织架构移除 |
| 收回门禁卡 | 前台/IT | 确认注销门禁权限 |
| 收回笔记本电脑 | IT管理员 | 确认设备归还 |

#### 需要审批后执行

| 操作 | 审批人 | 说明 |
|------|--------|------|
| 禁用VPN权限 | IT主管 | 确保无远程访问 |
| 禁用业务系统账号 | 部门主管 | 财务/ERP/CRM等 |
| 代码库权限移除 | 研发主管 | 保留代码所有权 |

### 3.4 离职数据处理

1. **邮件处理**
   - 将邮件转发至主管或接替者
   - 自动回复设置为"该邮箱已停用，请联系xxx@company.com"
   - 6个月后彻底删除

2. **文件处理**
   - 共享文件夹权限移除
   - 个人云盘文件：通知主管确认是否需要交接
   - 代码/文档归属公司，个人不得删除

## 4. 账号权限管理规范

### 4.1 权限原则

1. **最小权限原则**：员工只能访问工作必需的系统
2. **岗位绑定原则**：权限与岗位挂钩，调岗需重新申请
3. **定期审查原则**：每季度审查一次权限清单
4. **离职即回收原则**：离职当日全部回收

### 4.2 权限申请流程

```
申请人提交 → 主管审批 → IT审核 → 开通权限 → 通知申请人
```

- 普通权限：1个工作日内完成
- 敏感权限（财务/系统管理）：3个工作日内完成
- 紧急权限：需IT总监额外审批

## 5. 常见问题解答

### Q1: 新员工入职当天发现账号无法登录怎么办？

**解决方案：**

1. 检查是否收到IT发送的账号开通邮件
2. 确认密码是否正确（初始密码通常为 姓名首字母+工号+@）
3. 如密码错误，使用"忘记密码"功能重置
4. 紧急情况联系IT热线：800-XXX-XXXX

### Q2: 岗位调整后需要开通新系统权限，怎么处理？

**解决方案：**

1. 员工通过OA提交《系统权限申请表》
2. 注明需要开通的系统、原因
3. 主管审批后IT自动开通
4. 通常1个工作日内完成

### Q3: 离职员工还有工作邮件需要处理？

**解决方案：**

1. 主管向IT提交邮件处理申请
2. IT将邮件转发至指定邮箱
3. 自动回复设置为停用提示

### Q4: 门禁卡丢失怎么办？

**解决方案：**

1. 立即联系前台和IT报备
2. IT后台禁用丢失的门禁卡
3. 重新制作新门禁卡（工本费20元）
4. 如有安全隐患，更换门禁密码

### Q5: 多人共用账号可以吗？

**解决方案：**

- **严格禁止！** 每人必须有独立账号
- 共用账号导致的问题无法追溯
- 如因工作需要临时共享文件，使用共享邮箱或共享文件夹
