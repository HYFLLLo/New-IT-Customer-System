## 1. 网络问题快速诊断流程

### 1.1 快速诊断流程图

```
网络连接异常？
        ↓
┌─────────────────────────┐
│ 检查网络指示灯            │
│ 电脑右下角网络图标什么颜色？│
└─────────────────────────┘
        ↓
    ┌───┴───┐
    ↓       ↓
  红色/叉   黄色感叹号
    ↓           ↓
  无网络     有网络但慢
    ↓           ↓
  按步骤2排查  按步骤3排查
```

### 1.2 快速自检命令

在电脑的**命令提示符**（Win+R 输入 cmd）中执行以下命令：

```bash
# 检查本地网络配置
ipconfig /all

# 检查网络连通性（测试DNS解析）
ping www.baidu.com

# 测试网关连通性
ping 192.168.1.1

# 刷新DNS缓存
ipconfig /flushdns

# 重置网络（管理员权限）
netsh winsock reset
```

## 2. 常见网络问题及解决方案

### 2.1 问题一：无法连接网络，网络图标显示红叉

**可能原因：** 网线未插好、网卡驱动异常、IP冲突

**排查步骤：**

#### 步骤1：检查物理连接

```
┌────────────────────────────────────────┐
│  1. 检查网线是否插紧                    │
│     - 从电脑和墙上插座都检查            │
│     - 重新插拔网线                      │
│                                        │
│  2. 检查墙上网络插座指示灯              │
│     - 绿色常亮 = 正常                   │
│     - 橙色闪烁 = 有数据流动             │
│     - 不亮 = 网络故障                   │
│                                        │
│  3. 尝试更换网线                        │
│     - 可能是网线老化                    │
└────────────────────────────────────────┘
```

#### 步骤2：检查网卡状态

1. 右键点击"此电脑" → "管理" → "设备管理器"
2. 找到"网络适配器"
3. 检查是否有黄色感叹号 ⚠️

**解决方案：**
- 如有感叹号：右键更新驱动，或使用驱动精灵
- 如网卡被禁用：右键"启用设备"

#### 步骤3：检查IP配置

1. Win+R 输入 cmd，输入 `ipconfig`
2. 查看IPv4地址

```
正常情况：
IPv4地址: 192.168.x.x （非169.254.x.x）
子网掩码: 255.255.255.0
默认网关: 192.168.x.1

异常情况：
IPv4地址: 169.254.x.x  ← 表示未获取到IP
```

**解决方案：**
```bash
# 释放并重新获取IP
ipconfig /release
ipconfig /renew
```

### 2.2 问题二：网络连接正常但无法上网

**可能原因：** DNS解析故障、浏览器设置问题、代理设置错误

**排查步骤：**

#### 步骤1：ping 测试

```bash
# 测试DNS解析
ping www.baidu.com

# 结果分析：
# - "来自 220.181.38.xxx 的回复" = DNS正常
# - "请求超时" = 可能是DNS或防火墙问题
# - "Ping 请求找不到主机" = DNS故障
```

#### 步骤2：更换DNS服务器

如果ping不通，尝试更换DNS：

1. 打开"网络和Internet设置"
2. 点击"更改适配器选项"
3. 右键网卡 → 属性 → 双击"Internet协议版本4"
4. 设置DNS为：
   ```
   首选DNS服务器: 114.114.114.114
   备用DNS服务器: 8.8.8.8
   ```
5. 确定后重新ping测试

#### 步骤3：检查浏览器

- 检查浏览器是否设置了代理
- 尝试使用其他浏览器
- 清除浏览器缓存（Ctrl+Shift+Delete）

---

### 2.3 问题三：网络经常掉线不稳定

**可能原因：** 网卡驱动过旧、网线质量差、交换机端口故障、网络攻击

**排查步骤：**

#### 步骤1：更新网卡驱动

1. 打开设备管理器
2. 找到网络适配器
3. 右键"更新驱动程序"
4. 选择"自动搜索更新的驱动程序软件"

#### 步骤2：检查网线

- 检查网线是否有折弯、破损
- 更换一条已知正常的网线测试
- 优先使用六类网线(CAT6)

#### 步骤3：检查网络设备端口

- 尝试将网线插到其他端口
- 同一端口别人能用吗？
- 记录是单个端口还是所有端口都有问题

## 3. 网络设备自检方法

### 3.1 办公室路由器/交换机

**设备位置：** 通常在弱电箱、机房、墙角柜子里

**自检步骤：**

```
┌────────────────────────────────────────┐
│           设备指示灯解读                │
├────────────────────────────────────────┤
│  PWR (电源灯)                          │
│  - 常亮绿色 = 正常                      │
│  - 不亮 = 电源故障                      │
│                                        │
│  SYS (系统灯)                          │
│  - 闪烁 = 运行正常                       │
│  - 常亮或快闪 = 启动中或故障             │
│                                        │
│  LAN口 (1-4)                            │
│  - 绿灯 + 闪烁 = 该端口有数据            │
│  - 绿灯 + 常亮 = 设备连上但无数据        │
│  - 不亮 = 该端口无连接或对方设备关机     │
│                                        │
│  WAN口 (外网)                           │
│  - 闪烁 = 外网连接正常                   │
│  - 不亮 = 外网断开                       │
└────────────────────────────────────────┘
```

**重启设备：**
1. 拔掉电源线
2. 等待30秒（让电容放完电）
3. 重新插上电源
4. 等待2-3分钟设备完全启动

⚠️ **注意：** 重启会影响整个办公室网络，请提前告知同事！

### 3.2 企业级AP（无线接入点）

**设备位置：** 天花板、墙上

**故障表现：**
- 某区域突然搜不到WiFi
- WiFi信号弱或不稳定

**自检：**
1. 检查AP的PWR灯和LAN灯
2. 重启AP（拔电源等30秒再插）
3. 检查PoE交换机对应端口

## 4. WiFi连接问题

### 4.1 搜不到公司WiFi

**解决方案：**

| 检查项 | 操作 |
|--------|------|
| WiFi开关 | 确认电脑WiFi开关已打开（有些笔记本有物理开关） |
| 飞行模式 | 确认没有开启飞行模式 |
| 距离 | 是否在WiFi覆盖范围内（太远信号弱） |
| 刷新 | 点击"刷新"网络列表 |

### 4.2 连接WiFi提示"验证失败"

**可能原因：** 密码错误、802.1X认证失败、证书过期

**解决方案：**

1. 确认WiFi密码（联系IT获取）
2. 尝试"忘记此网络"后重新连接
3. 检查是否需要手动输入用户名密码（企业WiFi）
4. 确认证书是否过期（联系IT更新证书）

### 4.3 WiFi已连接但无法访问网络

**解决方案：**

```bash
# 1. 先测试有线网络是否正常

# 2. 忘记网络重新连接
#    设置 → WiFi → 点击已连接的网络 → "忘记"

# 3. 重启WiFi适配器
netsh interface show interface
netsh interface set interface "Wi-Fi" disable
netsh interface set interface "Wi-Fi" enable

# 4. 检查IP获取方式
#    应设置为"自动获取IP地址"
```

## 5. VPN远程访问

### 5.1 VPN连接常见错误

| 错误代码 | 含义 | 解决方案 |
|---------|------|----------|
| 619 | 无法连接到VPN服务器 | 检查网络，或VPN服务未启动 |
| 720 | PPP协议错误 | 重启电脑，更新VPN客户端 |
| 800 | 无法连接到VPN服务器 | 检查服务器地址，或端口被封 |
| 809 | 服务器无响应 | 检查防火墙设置，或服务器宕机 |
| 138 | 域名前缀错误 | 联系IT确认正确的VPN域名 |

### 5.2 VPN连接步骤（Windows自带）

```
1. 打开"设置" → "网络和Internet" → "VPN"
2. 点击"添加VPN连接"
3. 填写信息：
   - VPN提供商：Windows(内置)
   - 连接名称：公司VPN
   - 服务器地址：vpn.company.com（联系IT获取）
   - VPN类型：L2TP/IPsec
   - 预共享密钥：联系IT获取
   - 用户名密码：你的企业邮箱账号密码
4. 点击"保存"
5. 选择VPN连接，点击"连接"
```

### 5.3 VPN已连接但无法访问内网

**检查步骤：**

```bash
# 1. 查看VPN分配的IP
ipconfig

# 正常VPN获取的IP应该是 10.8.x.x 或 172.16.x.x

# 2. 测试内网地址
ping 10.8.0.1  # VPN网关
ping 192.168.1.100  # 公司内网服务器（联系IT获取地址）

# 3. 查看路由表
route print
# 检查是否有内网网段的路由指向VPN适配器
```

## 6. 网络安全故障

### 6.1 突然无法访问某个网站

**可能原因：** 网站被防火墙拦截、DNS污染、IP被封

**解决方案：**

1. 检查是否所有网站都打不开（判断是全局还是局部）
2. 尝试使用手机4G网络访问（判断是公司网络问题还是网站问题）
3. 联系IT确认该网站是否在公司黑名单中

### 6.2 收到"网络被监控"的警告

**解释：** 公司IT部门部署了网络行为管理系统，会对访问敏感网站、下载大文件等行为进行监控和提示。这是正常的安全措施。

**解决方案：**
- 如需解除警告，联系IT说明情况
- 如确属误报，IT会将网站加入白名单

## 7. Escalation流程

### 7.1 问题升级标准

| 级别 | 描述 | 响应时间 | 升级对象 |
|------|------|----------|----------|
| P1 | 全楼层/整栋断网 | 5分钟 | IT运维总监 |
| P2 | 单个部门断网 | 15分钟 | IT运维组长 |
| P3 | 单人网络故障 | 1小时 | IT值班人员 |
| P4 | 网络慢、不稳定 | 4小时 | IT工单队列 |

### 7.2 升级前准备

联系IT时，请提供以下信息：

1. **姓名+工号+部门**
2. **故障现象**（详细描述）
3. **故障时间**（什么时候开始的）
4. **已尝试的操作**（如重启、换端口等）
5. **影响范围**（只有自己，还是多人）

### 7.3 IT联系方式

| 联系方式 | 号码 | 用途 |
|---------|------|------|
| IT服务热线 | 800-XXX-XXXX | 日常咨询、故障申报 |
| 紧急故障 | 189-XXXX-XXXX | P1级紧急故障 |
| IT邮箱 | it-support@company.com | 非紧急问题、工单 |
| IT微信群 | IT服务支持群 | 快速响应 |
