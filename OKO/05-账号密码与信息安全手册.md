```
┌─────────────────────────────────────────────────────────────────┐

│   │   密码泄露导致的后果：                                   │   │
│   │                                                          │   │

│   │      - 冒充身份发送邮件                                   │   │
│   │      - 窃取通讯录联系人                                   │   │

**常见密码安全问题：**

| 问题 | 风险等级 | 占比 |
|------|---------|------|
| 多平台使用同一密码 | 🔴 极高 | 65% |
| 使用弱密码（如123456） | 🔴 极高 | 18% |

| 与同事共享密码 | 🔴 极高 | 12% |

## 2. 密码设置规范

### 2.1 公司密码策略

**密码必须满足以下要求：**

```
┌─────────────────────────────────────────────────────────────────┐
│                      密码复杂度要求                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【长度要求】                                                   │
│  • 最少 8 位字符（建议 12 位以上）                            │
│                                                                  │
│  【字符类型】至少包含以下三种：                                   │
│  ✅ 大写字母（A、B、C...）                                      │
│  ✅ 小写字母（a、b、c...）                                      │
│  ✅ 数字（0、1、2、3...）                                      │
│  ✅ 特殊字符（!、@、#、$、%、^...）                            │
│                                                                  │
│  【禁止使用】                                                   │
│  ❌ 纯数字/纯字母                                              │
│  ❌ 键盘连续字符（qwerty、asdf）                                │
│  ❌ 个人信息（生日、手机号、姓名首字母）                         │
│  ❌ 公司名称、公司邮箱前缀                                        │
│  ❌ 常见单词（password、admin、welcome）                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 强密码示例

**好的密码示例：**

| 密码 | 强度 | 说明 |
|------|------|------|
| `P@ssw0rd123!` | 🟡 中等 | 8位，含特殊字符 |
| `GaoTie2024#BeiJing` | 🟢 强 | 16位，含有意义的中文拼音 |
| `Tr0ub4dor&3` | 🟢 强 | 14位，随机构成 |
| `correct horse battery staple` | 🟢 强 | 4个随机单词（ passphrase）|

**不好的密码示例：**

| 密码 | 问题 |
|------|------|
| `12345678` | 太简单，纯数字 |
| `password!` | 常见词 |
| `Zhangsan2024` | 包含姓名 |
| `Company123!` | 包含公司名 |

### 2.3 密码管理建议

**推荐使用密码管理器：**

| 软件 | 说明 | 费用 |
|------|------|------|
| 1Password | 跨平台同步 | 公司已采购授权 |
| LastPass | 浏览器插件方便 | 免费版可用 |
| Bitwarden | 开源免费 | 免费 |
| 系统自带 | iCloud钥匙串/Windows Hello | 免费 |

**密码管理器的优势：**
- ✅ 一个主密码解锁所有密码
- ✅ 自动生成强密码
- ✅ 自动填充登录表单
- ✅ 跨设备同步

## 3. 常见账号密码重置

### 3.1 企业邮箱密码重置

**方法一：自助找回（推荐）**

```
1. 访问邮箱登录页
   - 企业微信邮箱：mail.company.com
   - 或通过企业微信客户端

2. 点击"忘记密码"

3. 输入企业邮箱地址

4. 选择验证方式：
   - 手机验证码（绑定手机号时可用）
   - 邮箱验证码（绑定备用邮箱时可用）
   - 人工申诉

5. 设置新密码

6. 使用新密码登录
```

**方法二：联系IT重置**

```
适用场景：无法通过自助方式找回

联系IT：
- IT服务热线：800-XXX-XXXX
- IT邮箱：it-support@company.com
- IT微信群

验证身份：
- IT会要求提供：姓名、工号、部门
- 可能需要主管确认

重置后：
- IT会发送临时密码至你的手机
- 首次登录后必须修改密码
```

### 3.2 OA系统密码重置

```
重置流程：

1. 打开OA登录页
   地址：oa.company.com

2. 点击"忘记密码"

3. 输入工号和注册手机号

4. 接收短信验证码

5. 设置新密码

6. 登录并完善个人信息（首次）

如手机号已变更：
→ 联系HR更新手机号
→ 再进行密码重置
```

### 3.3 各类系统密码规则

| 系统 | 密码格式要求 | 重置方式 |
|------|-------------|----------|
| 企业邮箱 | 8位以上，含特殊字符 | 自助/IT |
| OA系统 | 6位以上纯数字 | 自助/HR |
| ERP系统 | 8位以上，含字母数字 | IT |
| 财务系统 | 复杂密码，季度强制更换 | IT+主管 |
| GitLab | 8位以上，含特殊字符 | IT |
| VPN | 8位以上，季度强制更换 | IT |

## 4. 账号安全最佳实践

### 4.1 多因素认证（MFA）

**强烈建议开启MFA！**

```
┌─────────────────────────────────────────────────────────────────┐
│                    认证方式安全等级对比                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  🔴 单因素认证（仅密码）                                        │
│     风险：密码泄露即被盗                                         │
│                                                                  │
│  🟡 双因素认证（密码+手机验证码）                               │
│     推荐：企业邮箱、OA系统                                       │
│                                                                  │
│  🟢 三因素认证（密码+手机验证码+设备确认）                       │
│     最安全：财务系统、核心业务系统                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**开启MFA的系统：**

| 系统 | 当前MFA状态 | 建议 |
|------|------------|------|
| 企业邮箱 | ✅ 已开启 | 保持开启 |
| OA系统 | ✅ 已开启 | 保持开启 |
| 财务系统 | ✅ 已开启 | 保持开启 |
| VPN | ❌ 未开启 | **强烈建议开启** |
| GitLab | ❌ 未开启 | 研发人员建议开启 |

### 4.2 定期更换密码

**公司强制要求：**

| 系统类型 | 更换周期 | 提醒方式 |
|---------|---------|---------|
| 财务系统 | 每季度 | 邮件+短信提醒 |
| 核心业务系统 | 每半年 | 邮件提醒 |
| 普通系统 | 每年 | 自助提醒 |

**密码历史记录：**
- 新密码不能与最近5次使用的密码相同
- 建议使用密码管理器记录密码历史

### 4.3 账号共享风险

```
⚠️ 严禁与他人共享账号！

为什么不能共享：
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  1. 无法审计：谁在什么时间做了什么操作？                      │
│                                                              │
│  2. 责任不清：出现问题无法追溯是谁的责任                      │
│                                                              │
│  3. 安全风险：共用账号=密码暴露给多人                         │
│                                                              │
│  4. 合规违规：审计要求一人一账号                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘

正确做法：
✅ 需要多人访问 → 申请创建子账号/权限
✅ 需要临时授权 → 使用权限借用流程
✅ 需要代操作 → 使用委托授权功能
```

## 5. 钓鱼邮件识别与防范

### 5.1 钓鱼邮件特征

**常见钓鱼邮件特征：**

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  【发件人伪装】                                                 │
│  ❌ 发件人名称看起来正常，但邮箱地址奇怪：                       │
│     "IT部门" <it-support@suspicious.com>                       │
│     "HR" <hr@company-help.com>                                 │
│                                                                 │
│  【紧急恐吓】                                                   │
│  ❌ "您的账号将在24小时内被冻结！"                             │
│  ❌ "检测到异常登录，请立即验证！"                             │
│  ❌ "您的快递有问题，请点击查看！"                             │
│                                                                 │
│  【伪造链接】                                                   │
│  ❌ 链接文字显示是公司地址，但实际指向钓鱼网站                   │
│     悬停查看真实链接（不点击！）                                │
│                                                                 │
│  【附件陷阱】                                                   │
│  ❌ 陌生人发送的压缩包/文档/ exe文件                             │
│  ❌ "发票"、"通知"、"简历"等诱惑性文件名                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 验证发件人真实身份

**三步验证法：**

```
当你收到可疑邮件时：

  第一步：检查发件人邮箱地址
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  正常：zhangsan@company.com                                 │
  │  异常：company_it_support@163.com                          │
  │                                                             │
  │  注意！攻击者经常使用与公司相似的域名                         │
  │  如：company-corp.com、company1.com、c0mpany.com            │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

  第二步：悬停查看链接真实地址
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  如果邮件中要求你点击链接：                                  │
  │  1. 鼠标悬停在链接文字上（不要点击！）                       │
  │  2. 查看浏览器底部显示的真实URL                              │
  │  3. 如果URL不是公司域名，立即报警！                          │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

  第三步：电话/内部通讯核实
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  如果邮件看起来像重要通知：                                   │
  │  1. 通过其他渠道（电话、企业微信）联系发件人确认             │
  │  2. 不要使用邮件中的联系方式                                 │
  │  3. 直接访问官方网站登录（不要通过邮件链接）                  │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘
```

### 5.3 钓鱼邮件应急处理

**如已点击钓鱼链接或输入密码：**

```
⚠️ 紧急！立即执行以下步骤：

┌─────────────────────────────────────────────────────────────┐
│  1. 立即断开网络（防止横向渗透）                              │
│     - 拔掉网线或断开WiFi                                     │
│     - 关闭被入侵的电脑                                       │
│                                                              │
│  2. 更改可能泄露的密码                                       │
│     - 如果输入了邮箱密码 → 立即修改邮箱密码                   │
│     - 检查是否有异常设置（转发规则、自动回复）                 │
│                                                              │
│  3. 报告IT安全团队                                           │
│     - 紧急热线：189-XXXX-XXXX（7x24）                        │
│     - 邮件：security@company.com                              │
│                                                              │
│  4. 配合安全调查                                             │
│     - IT可能需要检查你的电脑                                 │
│     - 提供详细的事件经过                                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**如收到钓鱼邮件：**

1. **不要点击任何链接**
2. **不要下载或打开任何附件**
3. **不要回复邮件**
4. **立即转发给 IT安全团队** (security@company.com)
5. **在举报后删除邮件**

## 6. 信息安全红线

### 6.1 绝对禁止的行为

```
╔═══════════════════════════════════════════════════════════════════╗

║     - 个人路由器、随身WiFi、4G路由器                               ║

║                                                                   ║

║     - 警惕社会工程学攻击                                           ║

### 6.2 违规处罚标准

| 违规级别 | 行为 | 处罚 |
|---------|------|------|

| 严重违规 | 过失导致数据泄露、严重违规 | 书面警告+绩效处罚+全公司通报 |

## 7. 数据安全与隐私保护

### 7.1 数据分类分级

```
┌─────────────────────────────────────────────────────────────────┐
│                      公司数据分级                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【绝密】⭐⭐⭐⭐⭐                                              │
│  • 核心技术资料、源代码                                          │
│  • 战略规划、并购资料                                           │
│  • 处罚标准：变更/删除需董事长审批                              │
│                                                                  │
│  【机密】⭐⭐⭐⭐                                                │
│  • 客户名单、合同价格                                           │
│  • 员工薪资、人事档案                                           │
│  • 处罚标准：变更/删除需部门VP审批                              │
│                                                                  │
│  【秘密】⭐⭐⭐                                                  │
│  • 内部会议纪要、工作流程                                        │
│  • 处罚标准：变更/删除需部门总监审批                            │
│                                                                  │
│  【内部】⭐⭐                                                    │
│  • 员工手册、一般制度                                            │
│  • 处罚标准：变更/删除需部门主管审批                            │
│                                                                  │
│  【公开】⭐                                                      │
│  • 对外宣传资料、官网内容                                        │
│  • 处罚标准：可直接发布                                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 敏感数据处理规范

**可以做的：**

```
✅ 在公司内网处理工作文件

✅ 使用公司邮箱发送工作邮件（注意核对收件人）

✅ 将文件加密后通过公司网盘分享

✅ 打印机密文件后立即取走并销毁

✅ 离职前将工作文件交接给接替者
```

**绝对不可以做的：**

```
❌ 用个人邮箱发送工作文件

❌ 用微信/QQ/钉钉私人聊天发送机密信息

❌ 将工作文件存入个人百度网盘/U盘

❌ 在公共场所大声讨论敏感信息

❌ 将含敏感信息的打印件直接扔垃圾桶
```

### 7.3 敏感文件销毁

**正确销毁方式：**

| 载体类型 | 销毁方式 | 说明 |
|---------|---------|------|
| 纸质文件 | 碎纸机 | 使用交叉碎纸模式 |
| U盘/硬盘 | IT统一消磁/粉碎 | 联系IT部门处理 |
| 光盘 | 物理粉碎 | 不可直接扔垃圾桶 |
| 手机/平板 | 恢复出厂设置+IT监督 | 交回IT部门 |
